in one line.

Stop copy-pasting the same database connection boilerplate into every project. next-mongo-connect handles it for you.

$npm install next-mongo-connect
Usage

Drop it in, done.

// app/api/route.js
import { NextResponse } from "next/server"
import dbConnect from "next-mongo-connect"
import User from "@/models/User"

export async function GET() {
  await dbConnect(process.env.MONGO_URI)
  const users = await User.find()
  return NextResponse.json({ users })
}
// pages/api/users.js
import dbConnect from "next-mongo-connect"
import User from "@/models/User"

export default async function handler(req, res) {
  await dbConnect(process.env.MONGO_URI)
  const users = await User.find()
  res.json({ users })
}
Why

What it does

⚡
Connection caching
Reuses existing connections across hot reloads in dev and across serverless invocations in prod.
🔌
Zero config
Pass your connection string and go. No setup files, no wrappers, no boilerplate.
📦
Tiny footprint
Single file, no unnecessary dependencies. Just Mongoose under the hood.
🔀
Works everywhere
App Router, Pages Router, both. Same API, same behaviour.
